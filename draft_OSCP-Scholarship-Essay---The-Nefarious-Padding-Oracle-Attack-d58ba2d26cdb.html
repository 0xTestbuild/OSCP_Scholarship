Introduction

Encryption, as we all may know, is a method used to obfuscate (or hide) data. The Padding Oracle Attack is a method used to decode an encryption technique known as Cipher Block Chaining, or CBC for short. The CBC encryption method works very much like a chain: plain text is divided up into blocks, or “chain links”, and used to secure the next block or chain link. 

The logic behind the encryption method may seem complex, but it is actually very simple. You begin with some text. Technical specialists call this the “initialization vector”, but we will refer to it as the first link in the chain. This chain link, just like everything inside the computer, is represented with a combination of 1's and 0’s. We perform an operation called XOR on these bits with the next link in the chain (which will be the first block of the message we want to encrypt) before before sending it through another encryption method. The encryption produces the coded message to be used in the encryption of the next text block. It’s like locking the key to a safe inside of another safe! 

This encryption method, while effective, can be easily be broken if not coded correctly. Let us introduce the Padding Oracle attack! The encryption method is easy, but the attack is difficult. We start by going into a deep analysis and understanding of how the code works. Endless youtube videos, countless searches on Google, and many hours of code later, I finally got made it to a solution. 

Each block of the chain is 16 inches long, which required me learn about complex encryption standards. The cyber-security geeks may get a kick out of hiding simple concepts really love complex text and diagrams, but that was not going to stop me.

Padding is a concept that allows a message to reach the intended length. The CBC method only works when the chain link is divisible by 16. What if your message isn’t the the size needed to encrypt the message? This is where padding comes in. Padding essentially adds data to the end of a message in order to make it compatible with the length requirements to play nicely with the other portions of the code. 

Put simply, it provide a neat and organized layer of protection makes life a lot easier because it makes code more reusable and universal while making the data much easier to manage. This feature, however, introduces a vulnerability that can be exploited. 

The attack essentially changes the last bit in order to see what happens.
